验证方法和加密模式
大多数无线网络都使用某种安全设置。这些安全设置定义了验证（设备如何在网络上验证自己）和加密方法（数据在网络上传送时如何加密）。如果您在配置您的无线设备时没有正确地指定这些选项，将无法连接到无线网络。因此在配置这些选项时请务必小心。请参阅以下信息以了解您的无线设备支持哪种验证和加密方法。
验证方法
  • 开放系统
    允许无线设备不需要任何验证就可访问网络

  • 共享密钥（仅基础模式）
    所有要访问无线网络的设备共享一个预先设定的密钥。

  • WPA/WPA2-PSK
    访问无线网络的所有设备共享一个预先设定的密码组合。这个验证方法通过一个强大的加密方法可使您安全地通信。
    WPA/WPA2-PSK 仅用于无线基础架构网络。

  • LEAP (不适用于某些型号)
    Cisco® LEAP （轻度扩展验证协议） 是由 Cisco 系统有限公司开发的,它使用用户名和密码来进行验证。

  • EAP-FAST/PEAP/EAP-TTLS (不适用于某些型号)
    Cisco® EAP-FAST（扩展身份验证协议－通过安全隧道的灵活认证）/ PEAP（受保护的扩展身份验证协议）/EAP-TTLS（扩展身份验证协议－隧道传输层安全）使用用户ID和通行证．您可以使用CA证书来验证服务器证书．

  • EAP-TLS(不适用于某些型号)
    EAP-TLS（扩展身份验证协议－传输层安全）使用用户ID和客户端证书进行验证。您还可以通过使用CA证书验证SSL服务器证书。

内部验证方法(不适用于某些型号)
您需要根据所选项目指定其中一种内部验证方法。
  • 无
    MS-CHAPv2用于参数配置，而且也是一种对应第二阶段验证方法的EAP-FAST方法。

  • MS-CHAPv2
    MS-CHAPv2（Microsoft 质询握手身份验证协议 版本 2）是一种用于EAP-FAST/PEAP/EAP-TTLS的内部验证方法。

  • GTC
    GTC（通用令牌卡）是一种用于EAP-FAST/PEAP的内部验证方法。

  • MS-CHAP
    MS-CHAP（Microsoft 质询握手身份验证协议）是一种用于EAP-TTLS的内部验证方法。

  • CHAP
    CHAP（质询握手身份验证协议）是一种用于EAP-TTLS的内部验证方法。

  • PAP
    PAP（密码验证协议）是一种用于EAP-TTLS的内部验证方法。

加密方法
  • 无
    未使用过加密方法

  • WEP (有线等效协议)
    数据通过用户选择的一个安全密钥被传送和接收。

  • TKIP
    TKIP（临时密钥完整性协议）消除了由于恶意访问者使用手动分配的WEP密钥而引起安全性隐患的可能性。TKIP是自动生成的，而且要比WEP密钥长。

  • AES
    AES （高级加密标准）是由 Wi-Fi(R)权威认可的强大的加密标准。

  • CKIP
    这是Cisco 系统有限公司针对LEAP的初始密钥完整协议。
