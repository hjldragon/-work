TCP/IP Erweitere Einstellungen (Verkabelt)
Boot-Versuche
Ist die Methode nicht STATIC, so gibt dieses Feld an wie oft das/die im Feld 'Methode' vorgegebene/n Protokoll/e zum Abfragen der neuen Adressenparameter versucht werden. Wird nach dieser Anzahl an Versuchen keine neue Adresse erhalten, so werden die vorhandenen Werte verwendet.
RARP Boot-Einstellungen
Ist die Boot-Methode RARP (oder AUTO, was RARP beinhaltet) so zeigen diese beiden Markierungen an, ob RARP (das nur eine Adresse liefert) die Subnet Mask basierend auf diese Adresse einstellen soll, oder das Gateway, basierend auf den die RARP-Information bereitstellenden Host.
TCP-Zeitlimit
Wenn sich ein TCP-basierter Job für einen bestimmten Zeitraum im Leerlauf befindet (sendet keine Daten), wird er gelöscht.
Ein Wert von Null (0) deaktiviert diese Funktion und Jobs sind in ihrer Gültigkeit zeitlich unbegrenzt.
Die Zeit bis zum Löschen eines Jobs kann in Minuten festgelegt werden.
DNS-Servermethode
Wenn in Ihrem Netzwerk ein DHCP-Server zum Zuweisen von IP-Adressen verwendet wird, kann dieser so konfiguriert werden, dass er auch andere IP-Adressen, wie z.B. die Adresse des DNS-Servers automatisch vergibt.
AUTO (Standardeinstellung) - Wenn Ihr DHCP-Server so konfiguriert ist, dass er DNS IP-Adressen zuweist, bewirkt die Einstellung Auto, dass der DNS-Server die IP-Adressen automatisch vom DHCP-Server erhält.
STATIC - Wählen Sie diese Einstellung, wenn Sie die IP-Adressen des DNS-Servers manuell eingeben möchten.
Primäre DNS-Server-IP-Adresse
Wenn in Ihrem Netzwerk ein DNS-Server (Domain Name Service) verwendet wird, gibt dieses Feld die IP-Adresse des primären DNS-Servers an.
Sekundäre DNS-Server-IP-Adresse
Dieses Feld gibt die IP-Adresse des sekundären DNS-Servers an. Sie wird als Backup für die primäre DNS-Serveradresse verwendet. Falls der Primärserver nicht verfügbar ist, kontaktiert das Gerät den sekundären DNS-Server.
Gateway-Timeout
Das Gerät wartet nun darauf, dass die DNS/SMTP-Server gefunden werden, auf die über das Gateway (den Router) zugegriffen wird. Wenn die Server nicht innerhalb der vorgegebenen Zeit gefunden werden, tritt ein Zeitüberschreitungsfehler auf.
Die richtige Zeit muss je nach verwendetem Gateway (Router) eingestellt werden. Wenn ein Router an einer Wählverbindung verwendet wird, empfehlen wir mehr als 60 Sekunden. (Die Standardzeit ist 5 Sek.)
