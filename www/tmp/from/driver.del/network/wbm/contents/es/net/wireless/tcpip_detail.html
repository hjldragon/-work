TCP/IP Configuración avanzada (Inalámbrica)
Intentos de arranque
Si el método no es STATIC, este campo indica cuántas veces el protocolo(s) especificado por el campo de método se probará para obtener nuevos parámetros de direcciones. Si no se recibe ninguna nueva dirección después de este número de intentos, se utilizarán los valores de dirección existentes.
Configuración del arranque RARP
Si el método de arranque es RARP (o AUTO, que incluye RARP), estos dos indicadores muestran si RARP (que proporciona una dirección únicamente) debe establecer la máscara de subred basándose en esa dirección o la puerta de acceso a la dirección de la máquina que proporciona la información RARP.
Tiempo de espera TCP
Si un trabajo con base TCP está inactivo (no envía datos) durante el período de tiempo especificado, se terminará el trabajo.
Un valor de cero (0) desactiva esta función y los trabajos nunca agotarán el tiempo de espera.
El tiempo de espera puede especificarse en minutos seleccionando la unidad apropiada.
Método del servidor DNS
Si su red utiliza un servidor DHCP para asignar las direcciones IP a los dispositivos de la red, entonces es posible configurar su servidor DHCP para asignar automáticamente otras direcciones IP tales como la del servidor DNS.
*Si tiene alguna duda, contacte con su Administrador de la Red.
AUTO (Por defecto) - Si su servidor DHCP está configurado para asignar direcciones IP de DNS entonces AUTO recibirá automáticamente las direcciones IP de DNS desde el servidor DHCP.
STATIC - Seleccione este valor si desea asignar manualmente las direcciones IP del servidor DNS.
Dirección primaria IP del servidor DNS
Si su red está configurada para usar un servidor DNS (Domain Name Service: Servidor de Nombres de Dominio), este campo especifica la dirección IP del servidor primario DNS.
Dirección secundaria IP del servidor DNS
Este campo especifica la dirección IP del servidor DNS secundario. Se utiliza como respaldo a la dirección del servidor DNS primario. Si el servidor primario no está disponible, el dispositivo se pondrá en contacto con el servidor DNS secundario.
Tiempo de espera de la puerta de acceso
Este es el tiempo que el dispositivo esperará para encontrar los servidores DNS/SMTP que se acceden a través de la puerta de enlace (Router). Si no se puede encontrar el servidor dentro del tiempo establecido, se producirá un error de "Tiempo de espera".
Dependiendo de la puerta de enlace (Router), se deberá establecer el tiempo adecuado. Es recomendable establecer unos 60 segundos, si utiliza un router con acceso telefónico. (El tiempo predeterminado es 5 seg.)
