SNTP
Visión general
En esta página puede configurar los ajustes de SNTP.
Método del servidor SNTP
ESTÁTICO; el dispositivo usa el valor configurado en esta página para los campos del servidor y del puerto SNTP. AUTOMÁTICO; el dispositivo intentará usar DHCP para adquirir estos parámetros cuando se encienda o reinicie el servidor. El método de arranque en la página TCP/IP debe estar configurado en AUTOMÁTICO o DHCP para permitir DHCP
Dirección del servidor SNTP primario
Escriba en este campo el nombre de host o la dirección IP del servidor SNTP primario. (Por ej. sntp.ejemplo.net -o- 192.0.0.1). Si usa el nombre DNS del servidor SNTP, tiene que usar el FQDN (nombre de dominio plenamente cualificado), por ej. sntp.ejemplo.net.
Puerto del servidor SNTP primario
Especifique el número del puerto de destino del servidor SNTP primario al que se va a conectar.
Dirección del servidor SNTP secundario
Escriba en este campo el nombre de host o la dirección IP del servidor SNTP secundario. Sirve como copia de seguridad de la dirección del servidor SNTP primario. Si el servidor primario no está disponible, el dispositivo contactará con el servidor SNTP secundario.
Puerto del servidor SNTP secundario
Especifique el número del puerto de destino del servidor SNTP secundario al que se va a conectar.
Intervalo de sincronización
El dispositivo se sincronizará con el servidor SNTP automáticamente a intervalos de horas especificados.
Estado de la sincronización
Este campo muestra el estado de la sincronización con el servidor SNTP especificado.
Satisfactoria: La última sincronización se ha realizado correctamente.
Sincronización fallida: La última sincronización no se ha realizado.
SNTP sin especificar: Por una configuración de SNTP incorrecta, no se ha podido sincronizar cuando se encendió el dispositivo. O bien está desactivado el SNTP.
Sincronizando: El dispositivo está sincronizándose con el servidor SNTP. Dentro de un rato, pulse el botón Actualizar para comprobar si se han sincronizado satisfactoriamente.
NOTA :
El estado 'Sincronizando' dura 2 minutos como máximo. Si el estado es 'Sincronización fallida' o 'SNTP sin especificar', el servidor SNTP especificado no se puede encontrar. El servidor SNTP no existe o el host especificado no admite SNTP. Revise el nombre y el número de puerto del servidor SNTP.
