TCP/IP Geavanceerde instellingen (Draadloos)
Boot-pogingen
Als de methode niet STATIC is, geeft dit veld aan hoe vaak met de door het veld Methode gespecificeerde protocollen wordt geprobeerd om nieuwe adresparameters te krijgen. Als na dit aantal pogingen geen nieuw adres is ontvangen, worden de bestaande adreswaarden gebruikt.
RARP Boot-instellingen
Als de boot-methode RARP (of AUTO., wat ook RARP inhoudt) is, geven deze twee vlaggen aan of RARP (wat alleen een adres geeft) het subnetmasker moet instellen op dat adres, of de gateway moet instellen op het adres van de host die de RARP-informatie geeft.
Time-out TCP

Methode DNS-server
Als uw netwerk gebruik maakt van een DHCP server om een netwerk apparaat toe te wijzen met IP adressen, is het mogelijk om uw DHCP server zo te configureren dat het automatisch andere IP adressen, zoals de DNS server, toewijst.
*Neem bij twijfel contact op met uw netwerk beheerder.
AUTO (Standaard) - Als uw DHCP server ingesteld is om DNS IP adressen toe te wijzen, dan zal de keuze AUTOMATISCH de DNS IP adressen automatisch van de DHCP server ontvangen.
STATIC - Kies voor deze optie als u handmatig de DNS IP adressen wilt toewijzen.
IP-adres eerste DNS-server
Als uw netwerk is ingesteld om een DNS Server (Domain Name Service) te gebruiken, dan specificeert dit veld het IP adres van de primaire DNS server.
IP-adres tweede DNS-server
Dit veld toont het IP-adres van de secundaire DNS-server. Dit wordt gebruikt als een back-up van het adres van de primaire DNS-server. Als de primaire server niet beschikbaar is, zal het apparaat contact opnemen met de secundaire DNS-server.
Time-out voor gateway
Dit is de tijd die het apparaat wacht tot de DNS/SMTP-server is gevonden bij toegang via de gateway (router). Als de server niet wordt gevonden binnen de ingestelde tijd, treedt de fout 'Tijd verlopen' op.
Afhankelijk van de gateway (router) dient u de juiste tijd in te stellen. Voor een inbel-router raden wij 60 seconden aan. (de standaardtijd is 5 sec.)
