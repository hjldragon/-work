TCP/IP Impostazioni avanzate (Cablata)
Tentativi di avvio
Se il metodo è impostato su STATIC, questo campo indica quanti tentativi vengono effettuati per il protocollo o i protocolli specificati dal campo del metodo per ottenere nuovi parametri di indirizzi. Se non viene ricevuto nessun nuovo indirizzo dopo questo numero di tentativi, verranno utilizzati i valori degli indirizzi esistenti.
Impostazioni di avvio RARP
Se il metodo boot è RARP (o AUTO, che include RARP) questi due flag indicano se RARP (che fornisce solo un indirizzo) deve impostare la subnet mask in base a quell'indirizzo o al gateway all'indirizzo dell'host che fornisce le informazioni RARP.
Timeout TCP
Se un lavoro basato su protocollo TCP è fermo (non trasmette dati) per il periodo di tempo specificato , il lavoro verrà terminato. Un valore pari a zero (0) disabilita questa caratteristica ed i lavori non scadranno mai.
Il timeout può essere specificato in minuti, selezionando le unità appropriate.
Metodo server DNS
Se la rete utilizza un server DHCP per assegnare a dispositivi di rete indirizzi IP, è possibile configurare il server DHCP per assegnare automaticamente altri indirizzi IP come server DNS.
* In caso di dubbi, contattare l'Amministratore di Rete.
AUTO (Default) - Se il server DHCP è configurato per assegnare indirizzi IP DNS, selezionando AUTO l'indirizzo IP DNS verrà ricevuto automativamente dal server DHCP.
STATIC - Consente di inserire manualmente l'indirizzo IP del Server DNS.
Indirizzo IP server DNS primario
Se la rete è configurata per utilizzare un Server DNS (Domain Name Service) questo campo specifica l'indirizzo IP del Server DNS Primario.
Indirizzo IP server DNS secondario
In questo campo si specifica l'indirizzo IP del server DNS secondario, che si utilizza come backup dell'indirizzo del server DNS primario. Nel caso in cui il server primario non sia disponibile, il dispositivo contatta il server DNS secondario.
Timeout gateway
È il tempo di attesa della periferica per trovare il server DNS/SMTP a cui si accede tramite il gateway (router). Se il server non viene trovato entro il tempo impostato, viene generato un errore "Timeout".
A seconda del gateway (router), è necessario impostare il tempo corretto. Se si utilizza un router di accesso remoto, si consiglia un tempo superiore a 60 secondi. (Il tempo predefinito è 5 sec)
