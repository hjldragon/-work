SNTP
Panoramica
In questa pagina è possibile configurare le impostazioni SNTP.
Metodo server SNTP
STATICO: il dispositivo usa il valore impostato in questa pagina per il server SNTP e i campi della porta. AUTO: il dispositivo prova a usare DHCP per acquisire tali parametri quando il server si avvia o viene inizializzato. Il metodo BOOT nella pagina TCP/IP deve essere impostato su AUTO o DHCP per abilitare DHCP.
Indirizzo server SNTP principale
Questo campo consente di immettere il nome host o l'indirizzo IP del server SNTP principale. (Es. sntp.esempio.net o 192.0.0.1) Se si utilizza il nome DNS del server SNTP, occorre usare il nome FQDN (Fully Qualified Domain Name), ad esempio sntp.esempio.net.
Porta server SNTP principale
Specificare il numero della porta di destinazione del server SNTP principale al quale connettersi.
Indirizzo server SNTP secondario
Questo campo consente di immettere il nome host o l'indirizzo IP del server SNTP secondario. Viene utilizzato come backup per l'indirizzo server SNTP principale. Se il server principale non è disponibile, il dispositivo contatterà il server SNTP secondario.
Porta server SNTP secondario
Specificare il numero della porta di destinazione del server SNTP secondario al quale connettersi.
Intervallo di sincronizzazione
Il dispositivo si sincronizzerà automaticamente con il server SNTP ad intervalli alle ore specificate.
Stato sincronizzazione
In questo campo viene visualizzato lo stato della sincronizzazione con il server SNTP specificato.
Riuscita: l'ultima sincronizzazine è riuscita.
Non riuscita: l'ultima sincronizzazione non è riuscita.
SNTP non specificato: a causa di impostazioni SNTP non corrette, la sincronizzazione non è mai avvenuta dall'avvio del dispositivo. Oppure SNTP non è abilitato.
Sincronizzazione in corso: il dispositivo si sta sincronizzando con il server SNTP. Dopo qualche istante, premere Aggiorna per verificare che la sincronizzazione sia riuscita.
NOTA :
lo stato di sincronizzazione "Sincronizzazione in corso" si estenderà per 2 minuti al massimo. Se lo stato è "Non riuscita" o "SNTP non specificato", il server SNTP specificato non è stato rilevato. Il server SNTP non esiste o l'host specificato non supporta SNTP. Controllare il nome e il numero di porta SNTP.
