Service
Overview
This page allows you to modify the parameters associated with the service you selected from the service list.
Service Name
Specifies the name of the service. Normally, you do not need to modify this field.
Service Port
Specifies the physical port associated with the service.
Protocols
These check boxes determine the network protocols enabled for this service.
Filter
Specifies the data filter, if any, for this service.
0 is no filter, data passes unmodified.
1 is text substitution, by default CRLF for LF.
2 indicates an AppleTalk job. Filtering depends upon the hardware and options selected.
3 converts normal text into PostScript.
4 converts output data into PostScript Tagged Binary.
Control Strings
Specifies data strings to be sent at the start or end of a job.
Raw TCP Port
If non-zero, defines a TCP port to be used for connection to this service.
Bi-Directional
If checked, the service will send data back from the Device to the network. Normally you should not need to change this. In the case of an non-bidirectional PostScript Device, you can turn off bi-directional support to enable AppleTalk spoofing.
