TCP/IP Paramètres avancés (Câblé)
Essais d'amorçage
Si la méthode n'est pas STATIC, ce champ indique le nombre de tentatives du ou des protocoles spécifiés par le champ de méthode pour obtenir les paramètres d'une nouvelle adresse. Si aucune adresse n'est reçue après ce nombre de tentatives, les valeurs de l'adresse existante sont utilisées.
Paramètres d'amorçage RARP
Si la méthode d'amorçage est RARP (ou AUTO, ce qui inclut RARP) ces deux indicateurs spécifient si RARP (qui ne fournit qu'une adresse) doit définir le masque de sous-réseau en fonction de cette adresse ou configurer la passerelle à l'adresse du hôte donnant les informations RARP.
Délai d'inactivité TCP
Si un travail n'utilise pas le protocole TCP (n'envoie pas de données) pour une période de temps spécifiée, le travail se terminera. La valeur 0 désactive cette caractéristique, et les travaux ne seront jamais hors délai. Le délai doit être spécifié en minutes, en sélectionnant la valeur appropriée.
Méthode du serveur DNS
Si votre réseau utilise un serveur DHCP pour allouer les adresses IP aux différents périphériques, il est possible de configurer le serveur DHCP afin qu'il alloue d'autres adresses IP, telles que les serveurs DNS, automatiquement.
*En cas de doute contacter votre Administrateur Réseau.
AUTO (Par défaut) - Si votre serveur DHCP est configuré pour allouer les adresses IP DNS, le réglage AUTO permettra de recevoir automatiquement les adresses IP DNS depuis le serveur DHCP.
STATIC - Choisir cette sélection si vous souhaitez entrer manuellement les adresses IP des Serveurs DNS.
Adresse IP du serveur DNS primaire
Si votre réseau est configuré pour utiliser un serveur DNS (Domain Name Service), ce champ spécifie l'adresse IP du serveur DNS primaire.
Adresse IP du serveur DNS secondaire
Ce champ spécifie l'adresse IP du serveur DNS secondaire. Celle-ci est utilisée en guise de sécurité pour compléter l'adresse du serveur DNS primaire. Si le serveur primaire n'est pas disponible, l'appareil contactera le serveur DNS secondaire.
Délai d'attente pour la passerelle
Il s'agit du délai d'attente de l'appareil pour trouver les serveurs DNS/SMTP accessibles via la passerelle (routeur). Si le serveur est introuvable dans le délai prévu, une erreur de « temporisation » se produit.
Selon la passerelle (routeur), vous devez définir le délai approprié. Nous recommandons plus de 60 secondes, si vous utilisez un routeur par accès commuté. (Le délai par défaut est 5 sec)
