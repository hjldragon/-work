TCP/IP Дополнительные параметры (Проводной)
Попыток загрузки
Если задан метод, отличный от STATIC, это поле показывает, сколько раз протоколы, указанные в поле метода загрузки, пытаются получить новый адрес. Если после указанного количества попыток не удастся получить новый адрес, будет использоваться существующий адрес.
Параметры загрузки RARP
Если методом загрузки является RARP (или АВТО, который включает в себя RARP), эти два флажка указывают, должен ли RARP (который дает только адрес) устанавливать маску подсети на основе этого адреса или шлюз по адресу хоста.
Тайм-аут TCP
Если задание, отправляемое на печать по протоколу TCP, находится в состоянии бездействия (не отправляются никакие данные) в течение указанного периода времени, задание будет остановлено.
С помощью значения "ноль" (0) можно выключить эту функцию, и задание никогда не превысит тайм-аут.
Значение тайм-аута можно указать в минутах, выбрав соответствующую единицу измерения.
Метод сервера DNS
Если сеть использует сервер DHCP для выделения устройств с IP-адресами, то сервер DHCP можно настроить на автоматическое выделение других IP-адресов, например, сервера DNS.
*За помощью обратитесь к администратору сети.
AUTO (по умолчанию) - Если сервер DHCP настроен на выделение IP-адресов DNS, то метод AUTO будет автоматически получать IP-адреса DNS от сервера DHCP.
STATIC - Выберите этот метод, если нужно вручную ввести IP-адреса сервера DNS.
IP-адрес основного сервера DNS
Если сеть настроена на использование сервера DNS, то в этом поле указывается IP-адрес основного сервера DNS.
IP-адрес альтернативного сервера DNS
В этом поле указывается IP-адрес альтернативного сервера DNS. Он используется как резервный на случай недоступности основного сервера DNS. Если основной сервер DNS недоступен, устройство обратится к альтернативному серверу DNS.
Тайм-аут шлюза
Это время ожидания устройства при поиске сервера DNS/SMTP, доступ к которому осуществляется через шлюз (маршрутизатор). Если сервер не будет найден в течение установленного промежутка времени, появится сообщение об ошибке «Тайм-аут».
В зависимости от шлюза (маршрутизатора) следует задать правильное время ожидания. При использовании коммутируемого маршрутизатора рекомендуется установить более 60 секунд. (Время по умолчанию – 5 секунд.)
