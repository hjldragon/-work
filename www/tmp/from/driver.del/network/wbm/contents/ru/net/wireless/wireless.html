Беспроводной (Персональная)
Обзор
Устройства Brother обеспечивают как безопасное личное использование, так и безопасное использование на предприятии. Эта страница предназначена для настройки использования устройства дома или в личных целях.
Режим связи
Существует два типа беспроводных сетей: режим инфраструктуры и прямое подключение (Ad-hoc)
Инфраструктура
Главным элементом сети в режиме инфраструктуры является центральная точка доступа (обычно беспроводной маршрутизатор). Именно через эту центральную точку доступа устройство Brother принимает все задания.
Ad-hoc
В одноранговых сетях (называемых также ad hoc или P2P-сетями) отсутствует центральная точка доступа. Каждый беспроводной клиент связывается непосредственно с другим таким же. Если для беспроводного устройства установлен этот режим, оно получает все задания непосредственно с компьютера, отправляющего данные.
Имя беспроводной сети (SSID)
Идентификатор SSID (Service Set Identifier) указывает, какую беспроводную сеть нужно использовать.
Канал
В беспроводных сетях используются каналы. Можно использовать до 13 каналов. Однако в большинстве стран число доступных каналов ограничено.
Метод проверки подлинности и режим шифрования
Большинство беспроводных сетей используют тот или иной способ защиты.
Параметры безопасности определяют метод проверки подлинности (как устройство идентифицирует себя в сети) и шифрование (как шифруются пересылаемые через сеть данные).
Если при настройке беспроводного устройства эти параметры указаны неправильно, устройство не сможет установить соединение с беспроводной сетью.
Поэтому настраивать эти параметры нужно внимательно.
Чтобы узнать, какие способы проверки подлинности и шифрования поддерживает ваше беспроводное устройство для домашнего (личного) использования, смотрите информацию ниже.
Методы проверки подлинности
Это устройство поддерживает следующие способы проверки подлинности для домашнего (личного) использования:
- Открытая система
- Общий ключ (только в режиме инфраструктуры)
- WPA/WPA2-PSK
Открытая система
Беспроводным устройствам разрешен доступ к сети без проверки их подлинности.
Общий ключ (только в режиме инфраструктуры)
Все устройства используют для доступа к беспроводной сети общий, предварительно заданный секретный ключ.
WPA/WPA2-PSK
Все устройства используют для доступа к беспроводной сети общую, предварительно заданную секретную фразу-пароль.
Данный метод проверки подлинности обеспечивает безопасность соединения с помощью мощного метода шифрования.
WPA/WPA2-PSK может использоваться только в беспроводных сетях с режимом инфраструктуры.
Методы шифрования
Шифрование применяется для защиты самих данных. Данное устройство поддерживает следующие способы шифрования для домашнего (личного) использования:
Нет
Метод шифрования не используется.
WEP
Если используется метод WEP (Wired Equivalent Privacy), данные передаются и принимаются с применением секретного ключа.
TKIP
Протокол TKIP (Temporal Key Integrity Protocol) позволяет избежать предсказуемости, которая может применяться злоумышленниками для нахождения слабых мест системы безопасности, появляющихся в результате присвоенных вручную ключей WEP. Ключи TKIP генерируются автоматически. Они более длинные, чем ключи WEP, и поэтому предлагают более высокий уровень безопасности по сравнению со стандартными ключами WEP.
AES
AES (стандарт расширенного шифрования) является стандартом устойчивого шифрования, авторизированным Wi-Fi®.
Сетевой ключ
Для каждого метода защиты есть правила:
Открытая система/Общий ключ с WEP
Введите ключ WEP, который будет использоваться для доступа к сети. Этот ключ является 64-разрядным или 128-разрядным значением, которое должно быть введено в формате ASCII или в шестнадцатеричном формате (HEXADECIMAL).
64(40)бит ASCII
Используются 5 текстовых знаков.
например: Hello (заглавные и строчные буквы различаются)
64(40)бит шестнадцатеричный
Используются 10 знаков шестнадцатеричных данных.
например: 71f2234aba
128(104)бит ASCII
Используются 13 текстовых знаков.
например:B318 Wirelesscomms (заглавные и строчные буквы различаются)
128(104)бит шестнадцатеричный
Используются 26 знаков шестнадцатеричных данных.
например: 71f2234ab56cd709e5412aa3ba
WPA/WPA2-PSK и TKIP/AES
Используется предварительно заданный общий ключ (PSK) длинной более 7, но менее 64 символов.
