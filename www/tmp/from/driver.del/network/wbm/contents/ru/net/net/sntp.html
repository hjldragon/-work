SNTP
Обзор
На этой странице можно настроить параметры SNTP.
Метод сервера SNTP
СТАТИЧЕСКИЙ: устройство будет использовать значения полей SNTP-сервера и порта, заданные на этой странице. АВТО: устройство попытается использовать DHCP для получения этих параметров при запуске или перезагрузке сервера. Для включения DHCP на странице TCP/IP должен быть установлен метод загрузки «Авто» или DHCP.
Адрес первичного сервера SNTP
Это поле предназначено для ввода имени узла или IP-адреса первичного сервера SNTP. (Например sntp.example.net или 192.0.0.1). Если используется имя DHS сервера SNTP, необходимо использовать FQDN (полностью уточненное имя домена), например, sntp.example.net.
Порт первичного сервера SNTP
Укажите номер порта назначения первичного сервера SNTP, с которым будет установлено соединение.
Адрес вторичного сервера SNTP
Это поле служит для ввода имени хоста или IP-адреса дополнительного SNTP-сервера. Оно используется в качестве резерва для адреса основного SNTP-сервера. Если основной сервер недоступен, устройство обратится к дополнительному SNTP-серверу.
Порт вторичного сервера SNTP
Укажите номер порта назначения вторичного сервера SNTP, с которым будет установлено соединение.
Интервал синхронизации
Устройство будет автоматически выполнять синхронизацию с сервером SNTP через указанный интервал времени в часах.
Состояние синхронизации
В этом поле отображается состояние синхронизации с указанным SNTP-сервером.
Успешно: последняя синхронизация завершилась успешно.
Не удалось синхронизировать: последнюю синхронизацию выполнить не удалось.
Не указан SNTP: из-за неверных настроек SNTP синхронизацию ни разу не удалось выполнить с момента включения устройства. Или же протокол SNTP отключен.
Синхронизация: устройство выполняет синхронизацию с SNTP-сервером. Через некоторое время нажмите кнопку «Обновить», чтобы убедиться, выполнена ли синхронизация успешно.
ПРИМЕЧАНИЕ .
Состояние синхронизации "Выполняется синхронизация" сохраняется до 2 минут. Если указывается состояние "Не удалось синхронизировать" или "Не указан SNTP", значит, не удается обнаружить сервер SNTP. Сервер SNTP не существует или указанный узел не поддерживает SNTP. Проверьте имя сервера SNTP и номер порта.
